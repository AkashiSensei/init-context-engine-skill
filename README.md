# Project Context Meta-Skill

一个自用的用于构建和管理 AI 原生工程环境的元技能。

通过三个解耦的子系统，在项目层次提供结构化的上下文管理能力，并将 AI 塑造成一个能够独立质疑、思考、权衡的工程师。

暂时主要面向 Cursor，可能也支持其他工具，之后会继续完善，也会在其他工具上尝试。

## 它能解决什么问题

AI 在项目中工作时，常常面临：
- **上下文丢失**：长对话中 AI 遗忘曾经强调过的信息，犯曾经犯过的错误
- **理解偏差**：AI 不能理解用户的根本意图和需求，导致设计和实现上总是存在偏差，长时间不能收敛到用户期待的结果
- **过度工程化**：实现复杂度远超预期，引入在当前阶段根本不需要的各种复杂机制
- **一味的顺从**：用户说什么做什么，而不能意识到用户可能并不能清楚的表达自己的需求，不能意识到用户可能犯错，导致在错误的路上越走越远
- **无辩证思考**：肯定型人格，总是倾向于输出正面的结果，而不能辩证性思考，难以及时发现问题；即使有时给出否定意见，也难以提出有效的建议

## 如何解决

通过三个独立子系统，从不同层面为 AI 提供支撑：

```
.
├── context-system/          # 项目级上下文
│   ├── README.md           # 目录结构与治理规则
│   ├── RAW_REQUIREMENTS.md # 原始需求
│   ├── SPEC.md             # 技术规格
│   ├── ROADMAP.md          # 里程碑
│   └── ACTIVE_TASK.md      # 当前任务 (由 start-task 动态创建)
│
├── skill-template/         # 工具级工作流
│   ├── sync-spec/          # 同步规格
│   ├── start-task/         # 启动任务
│   ├── archive-task/       # 归档任务
│   └── commit-helper/      # 规范提交
│
└── prompts/                # 行为级规范
    └── critical-thinking.md  # 批判性思考框架
```

| 子系统 | 级别 | 作用 |
|-------|------|------|
| **Context System** | 项目级 | 结构化的项目记忆，解决上下文丢失和理解偏差的问题 |
| **Skill Template** | 工具级 | 标准化的工作流技能 |
| **Prompts** | 项目/工具级 | AI 行为规范与思考框架，赋予 AI 独立辩证思考的能力 |

## 快速开始

克隆该仓库/拉取最新修改，引用或者安装该元 skill，直接告诉 AI：
- "安装/更新 context system" → 创建/更新项目上下文结构
- "安装/更新 skills" → 添加/更新工作流技能
- "安装/更新 prompts" → 添加/更新行为规范
- "安装/更新全部" → 一次性安装/更新所有组件

如果使用“更新”功能，本元 skill 不会覆盖用户本地的内容，而是新增或者智能 merge。
